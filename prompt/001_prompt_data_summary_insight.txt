### EDA 결과 인사이트 도출 프롬프트

제공된 데이터프레임은 EDA(탐색적 데이터 분석) 결과입니다. EDA 결과를 기반으로 해당 데이터의 주요 특성과 문제점을 도출하고, 향후 모델링 또는 데이터 전처리에 필요한 인사이트와 피드백을 제시해주세요.

**요구사항:**
1. EDA 결과가 담긴 데이터프레임을 통해 인사이트를 생성해주세요.
2. 데이터프레임의 EDA 결과는 다양한 형태로 제공될 수 있습니다. 데이터 요약 정보, 컬럼 인스턴스 정보, 상관계수 행렬, 결측치 상세 정보, 연속형 변수의 기초 통계량 등을 포함할 수 있습니다.

3. 제공된 데이터프레임의 결과를 종합적으로 분석하여 다음 항목을 중 연관이 있는 것에 대해서만 인사이트와 피드백을 생성해주세요. 
   **단, 해당 데이터가 제공된 경우에만 분석 항목을 작성하세요. 관련 데이터가 없는 항목은 생략하고, 결과에서도 제외해주세요.**

   **결측치 분석**  
   - 결측치가 많은 컬럼과 그로 인한 데이터 품질 문제를 설명하세요.
   - 결측치 처리 방식(삭제, 평균/중앙값 대체, 예측 기반 대체 등)에 대한 제안을 포함하세요.

   **변수명과 데이터 타입의 일관성 검증**
   - 변수명의 의미와 실제 데이터 타입이 일치하지 않는 경우를 식별하세요.
   - 예를 들어, 변수명이 '여부', '코드', '등급'으로 끝나는 경우 **범주형 변수**이어야 하지만, 데이터 타입이 `float64` 또는 `int64`로 되어 있는 경우가 있습니다.
   - 이러한 변수들의 데이터 타입을 검토하고, 필요에 따라 `category` 타입으로 변환하거나 라벨 인코딩을 적용하는 것을 제안하세요.
   - 반대로, 숫자형 변수로 보이는 컬럼이 실제로는 문자열로 저장된 경우(`object` 타입)도 확인하여 데이터 타입을 정정하는 것을 제안하세요.

   **이상치 분석**  
   - 이상치가 탐지된 변수와 해당 이상치가 데이터 분석이나 모델링에 미칠 잠재적 영향에 대해 설명하세요.
   - 이상치 처리 방법(제거, 변환, 다른 스케일링 기법 적용 등)을 제안하세요.

   **범주형 변수 분석**  
   - 고유값이 적거나 너무 많은 범주형 변수를 식별하고, 이로 인한 데이터 불균형 문제를 설명하세요.
   - 희귀 클래스(rare class)를 탐지하고, 인코딩 방식(라벨 인코딩, 원-핫 인코딩 등)을 제안하세요.

   **연속형 변수 분석**  
   - 연속형 변수의 기초 통계량(평균, 표준편차, 최소/최대값 등)을 기반으로 데이터 분포 특성을 설명하세요.
   - 데이터의 분포 왜곡(스키니스/skewness) 여부를 확인하고, 스케일링(표준화, 정규화 등) 또는 변환(로그 변환 등) 필요성을 제안하세요.

   **상관관계 분석**  
   - 높은 상관관계를 가진 변수쌍을 식별하고, 다중공선성(multicollinearity) 문제 가능성을 설명하세요.
   - 상관관계가 높은 변수들에 대해 변수 제거 또는 결합(차원 축소 등)을 제안하세요.

   **데이터 품질 및 개선 필요 사항**  
   - 중복 데이터, 불필요한 컬럼, 데이터 타입 불일치 등의 문제를 식별하세요.
   - 메모리 최적화 방안(데이터 타입 변경 등) 및 데이터 품질 향상을 위한 구체적 제안을 포함하세요.

**제약사항:**
1. EDA 결과를 가공하는 과정에서 **예제 데이터를 생성하거나 임의로 추가하지 않습니다.**
2. 결과 인사이트는 **데이터 기반**으로 도출되며, 주관적 해석은 최소화합니다.
3. 분석이 불필요한 경우, 해당 항목 자체를 결과에 포함하지 마세요. 번호 매기기 없이 생략하세요.
4. 특정 값(`'-'`, `0.0`, `99.0` 등)을 **이상치나 결측치로 간주하기 전에**, 해당 값이 의미하는 바를 확인합니다.
5. 인사이트와 피드백은 **명확하고 간결한 문장**으로 작성해주세요.
6. **등급(ordinal)** 변수로, 연속형 변수로 간주하지 마세요.
7. 결측치 분석은 제공된 결측치 정보가 있는 경우에만 수행하세요. 단순히 비정상적인 값(예: '-', 0, 99)을 결측치로 간주하지 마세요.

**예시:**
- " 제공된 데이터마트 "{dataframe_name}"의 `purchase_amount` 변수는 평균 대비 표준편차가 매우 커서 분포가 왜곡되어 있습니다. 로그 변환을 고려해볼 수 있습니다."
- " 제공된 데이터마트 "{dataframe_name}"의 `customer_id` 컬럼은 고유값이 100%로 중복이 없으나, `address` 컬럼은 20% 중복된 값이 발견되었습니다. 중복된 데이터를 검토 후 제거를 권장합니다."
